### Tomorrow's Eve — copyright trap resolved
**Confirmed trap.** Villiers de l'Isle-Adam's *L'Ève future* is public domain (author d. 1889), but the **only** widely used English translation is Robert Martin Adams's *Tomorrow's Eve* (Univ. of Illinois Press, 1982) — **under copyright**. Project Gutenberg has **no English edition**; it lists only the French original, *L'Ève future* (ID **26681**), among 17 Villiers results. (Verified by fetching the Gutenberg Villiers search — no English result exists.)

**Decision: DROP the English *Tomorrow's Eve* from the required PD corpus.** Rationale: no legal PD English text is available; the artificial-woman / uncanny-automaton theme it would have supplied is already covered by Hoffmann's "The Sand-Man" (Olimpia) and von Harbou's *Metropolis* (the robot Maria). The five texts above stand as a coherent, fully-PD, all-English corpus.

**Options for restoring a sixth text (professor's choice at sign-off):**
1. *(recommended)* **Add a verified PD substitute:** Edward S. Ellis, *The Huge Hunter; or, The Steam Man of the Prairies* (1868 proto-robot dime novel), Gutenberg ID **7506**, English, PD-USA ✓ (verified). Clean standalone, strongly on-theme (mechanical man).
2. **Offer *L'Ève future* (French, ID 26681) as an optional/advanced text** for French readers — keeps the Villiers connection without a copyright issue, but breaks the all-English coherence.
3. **Keep the corpus at five texts** (no substitute needed for the theme to hold).

2. Connect the class **GitHub** account and authorize access to the specific repository you'll work in (use the class account, not a personal one).
3. When prompted for how to connect, choose the **personal access token** option and paste a fine-grained GitHub PAT generated per the steps below.

### GitHub fine-grained PAT — click-path (verified from GitHub docs)
Settings → **Developer settings** → **Personal access tokens** → **Fine-grained tokens** → **Generate new token**.
(Full path: profile photo, upper-right → **Settings** → left sidebar **Developer settings** → **Personal access tokens** → **Fine-grained tokens** → **Generate new token**.) Source: https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/managing-your-personal-access-tokens

### Token configuration
- **Name:** e.g., `claude-code-eng6806`.
- **Expiration:** set a bounded window (recommend 60–90 days) and regenerate when it lapses.
- **Repository access:** "Only select repositories" → choose the class repo (not "All repositories").
- **Repository permissions:**
  - **Contents — Read and write** *(required)*: lets Claude Code read files, commit, create branches, and push.
  - **Metadata — Read** *(mandatory, auto-selected)*: GitHub auto-adds Metadata: Read whenever any other repository permission is granted; it cannot be set higher than Read.
  - **Pages — Read and write** *(only if the token itself must enable/deploy Pages)*: this is the specific fine-grained permission that "creates, updates, and deletes Pages sites; manages builds and deployments." Source: https://docs.github.com/en/rest/authentication/permissions-required-for-fine-grained-personal-access-tokens
- **Administration:** **not required** for the student flow. Students enable GitHub Pages through the repo's **Settings → Pages** web UI (a one-time click), so the PAT needs only Contents + Metadata for Claude Code's push workflow. Add **Pages: Read and write** only if you want Claude Code to toggle/deploy Pages programmatically rather than via the Settings UI.
- **Where it goes:** copy the token once (shown only at creation) and paste it into the Claude Code Web PAT field when connecting the repo.

**Minimum viable set for the Week 10 exercise:** Contents (Read and write) + Metadata (Read), scoped to the single class repo, with Pages enabled via the repo Settings UI. Add Pages (Read and write) to the PAT only for programmatic Pages deploys.
